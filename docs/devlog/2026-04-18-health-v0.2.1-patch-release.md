# 2026-04-18 Health v0.2.1 Patch Release

## Context

The `health` suite was already released as `v0.2.0`, but the latest follow-up work exposed a release-quality gap:

- `private-doctor` already had a machine-checked reply contract
- `health-review`, `doctor-brief`, `health-reminders`, and `health-storage-feishu` still depended mostly on `SKILL.md` wording
- the public install docs still pointed to `v0.2.0`, even though the patch-level contract was being tightened

## Problem

That gap was small in scope but real in release quality:

- reply expectations across the suite were not enforced consistently
- release validation could miss regressions where outputs became too implicit or too short
- the next published install tag would otherwise lag behind the actual stable contract on `main`

## Decision

Ship this as a `patch` release, not a new feature release.

Why `patch`:

1. the public capability surface did not expand beyond health V1
2. the work mainly tightened reply-shape contracts and validation
3. the install surface stayed the same; only the stable tag and release proof moved forward

## Implementation

1. Added reply validators for `health-review`, `doctor-brief`, `health-reminders`, and `health-storage-feishu`.
2. Updated those runtime scripts so their JSON and markdown outputs expose the fields already required by the skill contracts.
3. Wired the new validators into install verification and suite acceptance.
4. Rebound release-facing install docs from `v0.2.0` to `v0.2.1`.
5. Added release notes for the new stable tag.

## Validation

Validated on the current worktree with:

```bash
python3 -m compileall health scripts/accept_health_suite.py scripts/verify_openclaw_health_agent_install.py
python3 scripts/validate_skill_boundaries.py
```

And a local end-to-end smoke flow that covered:

- archive session
- profile update
- doctor reply render + validation
- review generation + validation
- brief generation + validation
- reminder due-check + validation
- bundle export/restore + validation

## Outcome

- `v0.2.1` becomes the latest stable install target for the released health suite
- reply contracts are now machine-checked across the major health outputs, not only the doctor layer
- release-facing docs and release evidence are aligned again
