# 2026-04-18 GitHub 安装地址绑定到真实仓库

## 背景

前一轮已经把安装模型改成了 `GitHub tree URL + 安装提示词`，但文档里还停留在 `<owner>/<repo>` 和 `<tag>` 占位符。

## 问题

- 用户已经可以直接安装 skill，但 README 还不是可复制可用的真实地址
- 如果不把第一版 tag 定下来，正式安装示例仍然不稳定

## 决策

- 绑定真实仓库为 `redcreen/openclaw-skills`
- 把第一版稳定安装地址固定到 `v0.1.0`
- `main` 继续保留给开发安装示例

## 实施

1. 创建远端仓库并推送当前工作区。
2. 把所有公开文档中的 GitHub 安装示例替换成真实地址。
3. 保持生成器脚本的参数示例与真实仓库一致。
4. 更新控制面和开发计划，避免状态仍停留在“待绑定”。

## 结果

- 用户现在可以直接复制真实安装语句到 OpenClaw 对话框
- `health-archive` 和 `private-doctor` 都有稳定的 `v0.1.0` 安装地址
- 后续 release 的主要工作从“绑定仓库”变成“安装验收和自动化”
